# HEUS Strategy Thread — Source de continuité

## Objectif business
- Cible actuelle: 7 000€/mois
- Principe: revenue-first (leads -> trials -> payants -> MRR)

## Ligne stratégique (stable)
1. Acquisition qualifiée vers `/analyseur` (CTA principal)
2. Conversion funnel `/analyseur -> lead -> email -> trial -> payant`
3. Exécution quotidienne HEUS (daily war room) + arbitrage hebdo (weekly board)
4. Décisions fondées sur preuves: données internes + docs officielles + RETEX

## KPI nord-star
- Leads qualifiés/jour
- Trials/semaine
- Trial -> payant (%)
- MRR net hebdo

## Priorités actives (court terme)
- Fiabiliser conversion (pas trafic vanity)
- Éliminer faux positifs d'automatisation (SUCCESS réel vs BLOCKED)
- Garder une trace unique de chaque décision stratégique

## Protocole anti-perte du fil
- Toute décision stratégique doit être reflétée dans:
  - `MEMORY.md` (mémoire long terme)
  - `memory/HEUS-STRATEGY-THREAD.md` (fil directeur)
  - log opérationnel concerné (`daily-war-room.md`, `weekly-growth-review.md`, etc.)
- Chaque revue HEUS doit terminer par:
  - Ce qu'on continue
  - Ce qu'on arrête
  - Prochaine action exécutable (<24h)

## Reprise après arrêt/redémarrage
1. Lire `MEMORY.md`
2. Lire `memory/HEUS-STRATEGY-THREAD.md`
3. Lire les 2 derniers fichiers journaliers `memory/YYYY-MM-DD.md`
4. Lire `memory/daily-war-room.md` et `memory/weekly-growth-review.md`
5. Reprendre sur la "prochaine action exécutable" la plus récente
